ENTRADA DE PUNTOS

Cada jugador tendrá:

Input numérico de 3 dígitos
Teclado numérico mobile-friendly
Botón “SUMAR”
Botón rápido “30”

Reglas:

El primer equipo en llegar a 200 gana
Los puntos individuales se suman al total del equipo
REGLA ESPECIAL DEL BOTÓN “30”

El botón rápido “30” tendrá una lógica especial para evitar que un equipo gane automáticamente con ese botón.

COMPORTAMIENTO

Si el total actual del equipo es:

menor de 170:
el botón suma 30 normalmente.

Ejemplo:
140 + 30 = 170

Si el equipo tiene:

170 puntos o más

ENTONCES:

el botón “30” NO puede llevar el marcador a 200 o más.
automáticamente ajustará el total máximo permitido a 199.

Ejemplos:

170 + botón 30 = 199
180 + botón 30 = 199
195 + botón 30 = 199

IMPORTANTE:

El botón “30” nunca puede finalizar la partida.
Para ganar, el usuario debe introducir puntos manualmente usando el input numérico.
Esta validación debe aplicarse tanto visualmente como en backend/database.
LÓGICA TÉCNICA
if (teamScore >= 170) {
   teamScore = 199;
} else {
   teamScore += 30;
}
UX

Cuando ocurra este ajuste:

mostrar una pequeña animación
mostrar tooltip o toast:
“El botón 30 no puede cerrar la partida. Máximo permitido: 199”

El botón debe sentirse fluido y moderno.
